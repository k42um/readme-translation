# readme-translation

Workflow for automatically translating `README.md` into English (or any other language supported by DeepL) using the DeepL API.

## feature

- Translation using [DeepL API](https://www.deepl.com/pro-api)
- Preserves Markdown formatting for headings, lists, code blocks, images, badges, etc.
- Free setting of source and target languages
- Automatic push of translated files to the repository

## examples showing the use (of a word)

```yaml
name: Translate README

on:
  push:
    branches:
      - main
    paths:
      - README.md
  workflow_dispatch:

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - uses: actions/checkout@v4

      - uses: kazuma/readme-translation@v2
        with:
          deepl_api_key: ${{ secrets.DEEPL_API_KEY }}
          input_file: README.md
          output_file: README.en.md
          source_lang: JA
          target_lang: EN-US
```

## option

| Input | Required | Default | Description
|---|---|---|---|
| `deepl_api_key` | **Yes** | - | DeepL API key. Save it in the repository secret. 
| `input_file` | no | `README.md` | Path of the source file. | no
| `output_file` | no | `README.en.md` | Output file path of the translation result. | no | `README.en.md` | output file path of translation results.
| `source_lang` | No | `JA` | Language code of the source language (`JA`, `ZH`, `DE`, etc.). If an empty string is specified, DeepL will auto-detect it. | no | no
| `target_lang` | No | `EN-US` | Language code of the target language (`EN-US`, `EN-GB`, `FR`, etc.). | No
| `commit_changes` | No | `true` | Whether to autocommit and push translation files. | no
| `commit_message` | no | `docs: update translated README (auto-translated)` | Message on auto-commit. | no
| `commit_user_name` | No | `github-actions[bot]` | The Git `user.name` to use for automatic commits. | no
| `commit_user_email` | No | `github-actions[bot]@users.noreply.github.com` | The Git `user.email` used for automatic commits. | no

## setup

1. Create an account on [DeepL API](https://www.deepl.com/pro-api). (basically free of charge)
2. After logging in, go to **Account → API Keys and Restrictions → Create Key** to obtain your API key and copy it.
3. In the repository you wish to use, go to **Settings → Secrets and variables → Actions → New repository secret** and add a secret named `DEEPL_API_KEY`.
4. Create a repository `.github/workflows/translate-readme.yml` and copy the following code so that the README will be automatically translated into English! (If you want to customize it, please refer to the examples and options above.)

```yaml
name: Translate README
on:
  push:
    branches: main
    paths: README.md
  workflow_dispatch:

jobs:
  translate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: k42um/readme-translation@v2
        with:
          deepl_api_key: ${{ secrets.DEEPL_API_KEY }}
```

## remarks
- If you change the language during the process, the original translated file will remain. If you do not need it, please delete it.
